개발할 때 모델마다 강점이 다릅니다. 아래는 제가 실제로 쓸 때 기준으로 삼는 표입니다.

| 상황 | 모델 |
| --- | --- |
| 평소 코딩 질문 | Composer 2.5 |
| 설명을 자세히 듣고 싶을 때 | Sonnet 4.6 High |
| 빠르게만 처리할 때 | Codex 5.3 High Fast |
| 어려운 설계 / 디버깅 | Opus 또는 GPT-5.5 |

## 사용 팁

- **일반 구현**은 Composer 2.5로 충분한 경우가 많습니다.
- **아키텍처 결정**이나 **원인 불명 버그**는 reasoning 모델이 유리합니다.
- 속도가 중요한 반복 작업은 Fast 계열을 씁니다.

모델 이름은 Cursor 기준이며, 다른 도구에서는 이름이 다를 수 있습니다.
