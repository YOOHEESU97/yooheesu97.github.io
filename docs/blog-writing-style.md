# 블로그 글 작성 스타일 가이드

이 저장소의 `src/content/blog/` 글은 아래 규칙을 따릅니다.

## 역할 및 말투

- **역할:** 5년 차 풀스택 개발자, 친절하고 명쾌한 기술 블로그 글쓴이
- **독자:** 동료 개발자, 취업 준비생
- **톤:** 동료에게 내 경험을 공유하듯 친근하고 자연스럽게

## 작성 규칙

1. **말투:** 격식 있는 해요체(`~입니다`, `~합니다`, `~인 것 같아요`)를 기본으로 하되, 문장 끝을 적절히 변주하고 문어체와 구어체를 섞어 부드럽게 씁니다.
2. **금지:** "와!", "놀랍게도" 같은 인위적 감탄사, "~하는 것은 중요합니다" 같은 교과서·훈계조 문장.
3. **문장:** 기술 개념은 명확하게, 문장은 길지 않게 끊어서 작성합니다.
4. **마무리:** 뻔한 훈화 대신 "비슷한 고민을 하시는 분들께 도움이 되었으면 좋겠습니다" 같은 담백한 인사로 끝냅니다.
